# 0611 進度

## 產品總覽 — 三個子分頁
- 睡美人：從睡美人工作表 + 編號價目表補缺，無等級預設 S
- 正常品：新增 API `normal-products`，掃編號價目表中非睡美人且非不續辦的品項
- 不續辦：沿用既有 `discontinued-products` API
- 預載：儀表板載入時背景抓取全部三種資料，切頁秒讀

## 成本讀取邏輯
- 僅搜尋成本類欄位（`成本` → `單片成本` → `成本價`），不再 fallback 到 `單價`
- 優先順序：**編號價目表 → 睡美人工作表**
- 新增 `getPriceCostMap()` / `getSleeperCostMap()` 輔助方法

## SKU 庫存下限
- 庫存 < 3 坪的產品不列入產品總覽計算（KPI 與列表皆排除）

## 客戶下探
- 儀表板前 10 大客戶按下去跳出 modal，顯示該客戶年度所有交易
- 欄位：日期、業務、編號、系列、數量、金額、備註
- 新增 `customer-detail` API（掃經銷銷售報表原始資料）

## 產品總覽排序強化
- 新增排序選項：依庫存坪數、依上次銷售日、依銷售頻率
- 排序方向按鈕 `↓` / `↑`，點擊切換升降階，所有選項通用
- 後端新增 `saleCount` 欄位（交易次數）

## Bug 修復
- `getSleeperProductOverview()` 多餘右大括號導致所有 API 500

## 部署流程
```sh
git add -A
git commit -m "描述本次異動"
git push
```
- push main → GitHub Actions 自動觸發 FTP 上傳至 cPanel `public_html/sleeping-beauty/`
- `api.php` 與 `index.php` 直接覆蓋 HEAD，不建新版本
- 無需 composer 或任何建置步驟
