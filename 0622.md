## 業務分組與客戶管理規則實施

### 已完成

#### 1. 客戶名稱歸一化
- **明確合併**
  - 信義星 + 太爾 → 掛「信義星」
  - 琮達 + 琮威 → 掛「琮威」

#### 2. 交易篩選
- 新增 `saleCount` 欄位追蹤每個客戶的交易筆數
- 篩選規則：**交易 < 15 筆的客戶不列入統計**（排除單一案件客戶）
- 效果：客戶數由 ~200+ 減至 173

#### 3. 業務分組合併
- 新增 `$salesMerge` 對應表：`陳育瑋 → 陳勁多`
- 應用位置：
  - `getCustomerAnalysis()` → 銷售統計中的 salesRepCounts
  - `getCustomerTimeline()` → 時間軸中的銷售紀錄業務名稱

#### 4. 出貨篩選（既有邏輯，已確認生效）
- 備註含「樣品」/「扣帶」自動跳過
- SKU 結尾 `-S`/`-S1` 自動跳過
- 數量 = 0 或金額 = 0 自動跳過
