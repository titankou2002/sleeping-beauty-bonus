## 庫存總額計算修正
- `getInventorySummary()` 原本只統計睡美人工作表內的 SKU，導致總額被低估
- 改為遍歷完整 `getStockMap()`，成本來源依序 `getSleeperCostMap() → getPriceCostMap()`，無成本則跳過

## meeting.php KPI 樣式
- 新增 `.kpi-pct` 樣式：成長紅、衰退綠、佔比金色，數字加大加粗
- 應用於 YOY 成長率與佔總業績（睡美人/專案）百分比
- 出貨分群卡片 meta 數據（家數、佔比、平均拜訪、單次拜訪產值）同步加粗標色

## 產品總覽 (index.php) 強化
- 新增關鍵字篩選（編號/系列/尺寸）
- 客戶銷售區塊改為 `<details>` 收合，預設不展開
- 新增「近6月平均月銷」分析欄位（`monthlySpeedPings`）
- 產品卡重新設計：精簡為兩行摘要（圖、編號·系列·尺寸、庫存片/坪、最後銷售日及距今天數），其餘資訊收進「更多資訊」下拉，手機版友善

## 不續辦商品誤入睡美人統計 — Bug 修正
- `getSleeperConfig()` / `getSleeperProductOverview()` 未檢查不續辦旗標
- 修正：`getSleeperProductOverview()` 中加入 `if (!empty($profile['isDiscontinued'])) continue;`

## Gemini 補貨建議
- 新增 `getProductRestockAdvisor($tab)`：僅針對「正常品」分頁，依庫存/月銷篩選候選品，呼叫 Gemini 產生 補貨/觀察/清庫/正常 建議
- 結果快取於 `cache/restock_{tab}.json`，重建銷售快取時自動清除
- 修正 Gemini 回應錯誤（空白/截斷）：加大 `maxOutputTokens`、移除 markdown code fence、items 數量 40→25、錯誤訊息附帶 `finishReason`
- 睡美人/不續辦不產生補貨建議（不會補貨），但仍顯示「距今未銷售天數」

## 產品歷史快照
- 新增 `recordProductHistory($tab, $products)`：每月記錄各 SKU 的庫存坪數、歷史銷售、MOS、近6月月銷、未售天數
- 新增 `getProductHistory($tab, $sku)` 供查詢

## 保留單快照（高雅瓷內部管理）
- 確認「保留單」工作表可直接從 SS_ID_MAIN（高雅瓷內部管理）讀取，欄位含 `編號`(=SKU)、`數量`等，1997 筆資料
- 新增 `getReservationMap()`：依 `編號` 統計各 SKU 保留數量(`reservedQty`)與保留筆數(`reservedCount`)
- 併入 `recordProductHistory()` 每月快照

## 產品生命週期分析（新功能）
- 新增 `getProductLifecycle($sku)`：
  - 首次/最新進貨日（來自編號價目表 `首次進貨日`/`最新進貨日`）、距今天數
  - 目前上架通路數與明細（來自版面上架清單）
  - 每月觀察記錄表（庫存、歷史銷售、月均銷、MOS、未售天數、保留筆數/數量）
- 新增 API `product-lifecycle`
- index.php：產品卡「更多資訊」內新增「📈 產品生命週期」按鈕，彈窗顯示上述資訊
